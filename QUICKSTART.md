# 快速开始指南

## 🚀 第一次使用

### 步骤1：了解项目结构
查看 `README.md` 了解完整的学习计划和项目说明。

### 步骤2：开始第一次学习

#### 如果是周末（2小时完整模考）：
1. 完成1套改革后真题
2. 立即记录错题：
   ```bash
   # 复制模板到对应目录
   cp templates/error_template.md errors/reading/你的错题文件名.md
   ```
3. 填写错题信息
4. 更新 `progress/test_scores.md` 记录成绩

#### 如果是平日（30-60分钟单项练习）：
1. 选择TPO单项或专项练习
2. 完成练习后记录错题
3. 更新 `progress/weekly_progress.md`

### 步骤3：请求AI分析
在错题文件中填写"需要AI分析的问题"部分，然后告诉我：
```
请帮我分析 errors/reading/你的错题文件名.md
```

## 📝 常用操作

### 记录错题
1. 复制 `templates/error_template.md`
2. 放到对应的 `errors/` 子目录
3. 重命名为有意义的文件名（如：`tpo45_reading_q12.md`）
4. 填写错题信息

### 更新进度
- **每周日**：更新 `progress/weekly_progress.md`
- **每次模考后**：更新 `progress/test_scores.md`

### 请求帮助
- 分析单个错题：`请帮我分析 [文件路径]`
- 分析本周错题：`请帮我分析本周所有的[题型]错题`
- 制定计划：`根据我的错题记录，帮我制定下周的学习重点`

## 💡 小贴士

1. **及时记录**：完成练习后立即记录错题，不要拖延
2. **文件命名**：使用清晰的命名，便于查找
3. **定期回顾**：每周查看进度文件，了解自己的进步
4. **保持简单**：不要过度复杂化，重点是坚持记录
